Avoid casts and develop a cast-free alternative whenever practical, especially dynamic_cast in performance-sensitive code.

 Feb 2, 2018

Postponing variable definitions as long as possible increases program clarity and improves progranm efficiency.

Item-25 Consider support for a non-throwing swap

 Feb 2, 2018

When std::swap would be inefficient for your type,provide a non-throwing swap member function, a non-member swap calling the member, and possibly a specialized std::swap for the case of classes (not templates).

Item-24 Declare non-member functions when type conversions should apply to all parameters

 Feb 2, 2018

If we need type conversions on all parameters to a function including the one pointed to by the this pointer, the function must be a non-member.

Item-23 Prefer non-member non-friend function to member function

 Feb 2, 2018

Prefer non-member non-friend functions to member functions for better encapsulation, packaging flexibility, and functional extensibility.

Item-22 Declare data members private

 Feb 2, 2018

Declaring data members private gives clients syntactically uniform access to data, affords fine-grained access control, allows invariants to be enforced, and offers class authors implementation flexibility.

Item-21 Don't return a reference when we have to return an object

 Feb 2, 2018

Never return a pointer or reference to a local stack object, a refenrence to a heap-allocated object, or a pointer or reference to a local static object if there is a change that more than one such object will be needed (item 4 provides a “counter” example that is reasonable in single-threaded environments).

Item-20 Perfer pass-by-reference-to-const to pass-by-value

Pass by reference-to-const is typically more efficient than pass by value and avoids the slicing problem.
